# GWS-emoji-2-task-sheet

## 概要
このスクリプトは、個人用のスプレッドシートに追記された「新しいデータ行」を自動的に見つけ出し、指定した時間ごと（定期）に共有用のスプレッドシートへ自動転記（コピー）するスクリプトです。

タイマーによる定期実行を採用しているため、全自動での同期が可能です。

---

## 前提
- このスクリプトはGoogle Workspace Studio上で動作するエージェントと連携します
  - Google Workspace Studioで、Googleチャットに特定の絵文字でリアクションした際に、スプレッドシートにチャットの内容が転記されるFlowを作ります
    - Step1:
      - When an emoji reaction is added
        - 任意の絵文字を設定します
    - Step2:
      - Add a row
        - マイドライブ内に作った、自分だけがアクセスできるスプレッドシートに、新しい行が追加される設定をします
        - 指定したシートのカラムごとに、追加するデータを入力、または変数を設定します
  - 動作確認を完了します

## 導入手順 (Setup)

### 1. 共有用スプレッドシートの準備
* 転記先の共有用スプレッドシートを用意します。
* 対象となる「シート名」と、URLに含まれる「スプレッドシートID」を控えておきます。

### 2. 個人用（転記元）スプレッドシートの準備
* 転記したいデータを入力するシートを用意します。
* スクリプトが**「転記が完了したか」を自動で記録する『ステータス列』を必ず一つ設けます**。
  （例: D列をステータス列とする場合、そこにスクリプトが自動で記録を行うため、手入力は行わないようにします）

### 3. スクリプトの設定
1. 個人用スプレッドシートのメニューバーから `拡張機能` > `Apps Script` をクリックします。
2. 開いたエディタにあるデフォルトのコードをすべて削除し、最新のGASのコードを貼り付けます。
3. コード内 `【設定部分】` の以下の項目をご自身の環境に合わせて書き換えてください。

```javascript
  const TARGET_SPREADSHEET_ID = "共有スプレッドシートのID";
  const TARGET_SHEET_NAME = "シート1"; // 共有先のシート名
  const SOURCE_SHEET_NAME = "シート1"; // 個人用のシート名
  const STATUS_COLUMN = 4;           // スクリプトが「済」と書く列（D列=4）
  const DATA_COLUMNS = 3;            // 送りたいデータの該当する列数（A~C=3）
  const START_ROW = 2;               // データの開始行（2行目スタート=2）
```
4. `保存`（ディスクのアイコン）を押します。

### 4. トリガー（定期実行）の設定
1. GASエディタ画面の左端にある「時計アイコン（トリガー）」を押します。
2. 画面右下の青いボタン「トリガーを追加」をクリックします。
3. 以下のように設定し、保存します。
   * 実行する関数を選択: `syncDataPeriodic`
   * 実行するデプロイ: `Head`
   * イベントのソースを選択: `時間主導型`
   * 時間ベースのトリガーのタイプ: `分ベースのタイマー` または `時間ベースのタイマー`
   * 時間の間隔: お好みで選択（例:`15分おき`など、同期頻度を指定）
4. Googleのアクセス権限承認画面が出た場合は、「詳細」から「安全ではないページに移動」を選択して「許可」を行ってください。

## 運用上の動作と注意ポイント（重要）
* **【重要】一部空欄のデータも同期されます**: 今回の実装では、たとえばA〜C列をデータ列としている場合、A列だけ入力されていてB列・C列が空白の行であっても、ステータス列が空欄であればそのまま（一部空白の状態で）転記が行われます。
* **空行の自動スキップ機能**: ただし、A列〜C列すべてが完全に空っぽの「無記入の行」については、スクリプトが「不要なゴミデータ」と認識し、同期をスキップします。
* **二重書き込みの防止**: 一度転記された行は、ステータス列に自動で「済」と入ります。この文字が存在する限り、同じ行が二度送られることはありません。
* **修正データの再送について**: 一度転記された（「済」になった）あとに、元シートで空欄だったB列などを埋めても、再度自動で送信されることはありません。送り直したい行がある場合は、手動で該当行の「済」の文字を `Del` キー等で消去すれば、次のタイマー時に最新の状態が再び転記されます。
